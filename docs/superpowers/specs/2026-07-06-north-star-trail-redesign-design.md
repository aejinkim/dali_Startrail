# 북극성 트레일 — 타임타이머 리디자인 컨셉

날짜: 2026-07-06
상태: 컨셉 확정 (구현 방식은 추후 결정)
대상: `~/Projects/dali_timetimer` (현 북극성 타이머)의 전면 리디자인

## 1. 컨셉 한 줄

"북극성을 향해 걷는 타임타이머." 모든 집중 세션은 곧 한 걸음이고, 앱을 열 때마다 내가 어디까지 왔는지 지도로 마주한다. 기존 앱의 핵심 질문 — "이 10분은 나의 북극성을 향해 가고 있는가?" — 를 텍스트가 아니라 화면 구조 자체로 답하게 만든다.

레퍼런스:
- 비주얼: Braun/Teenage Engineering 계열 레트로 하드웨어 (크림 바디 + 오렌지 노브) 및 다크 이브닝 다이얼
- 게임 요소: 토스 만보기 (캐릭터 + 산길 여정, 수집·보상 요소는 제외)

## 2. 핵심 결정 사항

| 항목 | 결정 |
|---|---|
| 산출물 범위 | 디자인 컨셉/스펙 우선, 코드 베이스 결정은 추후 |
| 게임 깊이 | 캐릭터 + 여정 시각화 (보상/수집 없음) |
| 북극성 개수 | 1개 (모든 프로젝트·세션이 하나의 별을 향함) |
| 진행률 기준 | 예상 세션 수 기반 |
| 플랫폼 | 모바일 폭 웹앱 (PWA 가능) |
| 비주얼 스타일 | 듀얼 모드 — 라이트(크림 브라운 하드웨어) + 다크(이브닝 다이얼), "전원을 켠다" 메타포 |
| 여정 시각화 | 등고선 페이퍼 맵 (크림 종이 + 등고선 + 오렌지 트레일) |
| 앱 구조 | 여정 맵이 홈, 하단 고정 카드로 원탭 세션 시작 |

## 3. 목표 구조와 진행률

계층: 북극성 1개 → 프로젝트 N개 → 태스크 N개 → 세션(포모도로).

태스크 생성 시 "예상 세션 수"를 입력하며, 모든 진행률이 여기서 파생된다.

- 태스크 진행률 = 완료 세션 ÷ 예상 세션
- 프로젝트 진행률 = 프로젝트 내 완료 세션 합 ÷ 예상 세션 합
- 북극성 진행률 = 전체 완료 세션 합 ÷ 전체 예상 세션 합

예상이 빗나가면 태스크의 예상 세션 수를 수정할 수 있다. 분모가 커지면 남은 길이 길어질 뿐, 이미 걸은 거리(완료 세션)는 보존된다.

## 4. 지형 은유 — 여정 맵 규칙

지도의 모든 요소는 데이터와 1:1 대응한다.

- 산(봉우리) = 프로젝트. 등고선 크기가 예상 세션 총량에 비례
- 캠프(체크포인트) = 태스크 완료 지점
- 정상 깃발 = 프로젝트 완료
- 오렌지 실선 = 걸어온 길, 점선 = 남은 길
- 북극성 = 하늘에 항상 떠 있는 최종 목적. 모든 봉우리를 넘으면 도달
- 캐릭터 = 현재 진행 중인 태스크의 위치에 서 있음

## 5. 화면 구성 (4개 + 온보딩)

1. **여정 맵 (홈)** — 상단에 북극성 이름 + 전체 진행률, 중앙에 등고선 지도, 하단 고정 "현재 태스크" 카드(태스크명, 세션 n/m, 재생 버튼). 재생 버튼이 앱에서 가장 중요한 버튼. 탭바: 여정 / 프로젝트 / 기록
2. **집중 모드** — 풀스크린 다크 다이얼. 탄(tan) 링이 타임타이머처럼 남은 시간에 비례해 줄어들고 레드 니들이 남은 시간을 가리킴. 하단에 트레일 미니바(이 세션이 끝나면 어디까지 가는지). 휴식 타이머는 같은 화면의 톤 변형
3. **프로젝트** — 프로젝트/태스크 CRUD, 예상 세션 수 편집, 세션 길이 프리셋(5/10/20분 등)
4. **기록** — 주간 분석 계승. 날짜별 세션 히트맵 + "이번 주 걸은 거리" 같은 여정 언어로 번역
5. **온보딩 리추얼** — 첫 실행 시 북극성을 정하는 의식. 어두운 화면에 별 하나를 띄우고 문장을 입력하면 지도가 밝아지며 산이 솟아오르는 연출

## 6. 세션 플로우 — "전원을 켠다"

1. 홈에서 재생 탭 → 크림 지도가 어두워지며 다이얼이 켜짐 (기기 부팅 메타포)
2. 세션 진행 — 링 감소 + 니들 스윕
3. 완료 → 지도가 다시 밝아지며 캐릭터가 실제로 몇 걸음 걷는 애니메이션
4. 태스크 완료 시 캠프 도달 연출, 프로젝트 완료 시 정상 깃발 연출
5. 세션 중 이탈 시 니들 정지. 포기하면 걸음 없음 — 부분 크레딧 없음 (단순함 유지)

## 7. 캐릭터

단순한 흰 몸통의 둥근 탐험가. 표정과 소품(오렌지 배낭)만으로 감정 표현. 상태 4개:

- 대기 — 홈에서 서 있음
- 걷기 — 세션 완료 후 전진 애니메이션
- 축하 — 캠프/정상 도달
- 낮잠 — 3일 이상 세션 없을 때 (죄책감이 아니라 귀여움으로 복귀 유도)

## 9. 기존 앱에서 계승 / 버리는 것

계승:
- 북극성 문구 리추얼 ("이 10분은 나의 북극성을 향해 가고 있는가?")
- 세션 길이 조절 (슬라이더 → 프리셋으로 단순화)
- 주간 분석 (기록 화면으로 이전)
- Supabase 인증 (구현 시 재활용)

버리는 것:
- 단일 스크롤 구조
- 파란색 계열 팔레트
- "오늘 할 일 3개" 별도 리스트 — 태스크 시스템에 흡수. 오늘 할 태스크는 지도 하단 고정 카드가 대신함

## 10. MVP 경계

MVP:
- 북극성/프로젝트/태스크 CRUD (예상 세션 수 포함)
- 세션 타이머 + 라이트↔다크 듀얼 모드 전환
- 여정 맵 (정적 SVG + 세션 완료 시 전진 애니메이션)
- 캐릭터 2상태 (대기/걷기)
- 로컬 저장 (localStorage)

이후 단계:
- 온보딩 연출 고도화 (별 → 산 솟아오르는 인트로)
- 캐릭터 4상태 (축하/낮잠 추가)
- 기록 화면 (히트맵 + 여정 언어 통계)
- 사운드 (기계식 틱, 완료 벨)
- 계정 동기화 (Supabase)
